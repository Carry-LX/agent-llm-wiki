# SFT后RL训练性能先降后升的机制解析

这是一道大模型 post-training 方向的高频面试题：SFT 后进行 RL 训练，为什么模型在测试集上的表现先下降后上升？DeepSeek-R1、GPT-4、Claude 等模型的训练流程都是 SFT → RLHF/RLVR，理解这个问题直接关系到对两个训练阶段本质差异的把握。

## 核心结论

RL 初期性能下降是因为 SFT 和 RL 的目标函数存在本质冲突，RL 会破坏 SFT 学到的特征对齐。后续性能回升则依靠两个机制：**权重空间的方向性重构**（奇异向量旋转）和**奖励方差驱动的隐式正则化**。最终模型在两个优化目标之间找到平衡点。

---

## 第一阶段：性能为什么下降

### SFT 与 RL 目标函数的不可解耦性

华为 2026 年发表的论文从数学上证明：**SFT 和 RL 两个训练阶段本质上是不可解耦的**。无论先做哪个，第二个阶段必然会破坏第一个阶段学到的东西。

具体原因：

- **SFT 目标**：最小化负对数似然（NLL），让模型精确匹配训练数据分布。本质是"背标准答案"。
- **RL 目标**：最大化期望奖励，只关心输出能不能拿高分。本质是"自由探索，结果导向"。

论文的定理指出：即使 SFT 已经收敛，RL 训练开始后，SFT 的交叉熵损失**必然上升**。实验证实 Qwen3-0.6B 在 RL 开始后，CE loss 立刻飙升，甚至超过 base 模型水平。

### 参数空间视角

SFT 把模型参数拉向特定方向，让模型记住训练数据分布特征。RL 不关心记住了什么，只关心能不能拿高奖励，会强行把参数往另一个方向推。

类比：SFT = 学生背标准答案，RL = 学生自由探索只要结果对。从背答案突然切换到自由探索，一开始必然不适应。

## 第二阶段：性能为什么回升

### 关键发现：奇异向量方向比奇异值大小更重要

对模型权重矩阵做 SVD 分解后发现：**模型的泛化能力不取决于奇异值的大小，而是取决于奇异向量的方向**。

- SFT 训练会让权重矩阵的主成分方向发生旋转，尤其是最大和最小奇异值对应的方向，导致泛化能力下降。
- RL 训练会把这些方向**再旋转回来**，尤其是浅层和深层网络。

关键实验证据：如果把 RL 训练后模型的权重矩阵奇异向量强制替换回 SFT 阶段的方向，OOD 性能立刻暴跌——Llama-3.2-11B 的 OOD 准确率从 16.2% 掉到 10.6%，损失超过三分之一。这说明 **RL 找到的新方向才是恢复泛化能力的关键**。

### 隐式正则化：奖励方差驱动的自适应学习率

RL 训练有一种天然的隐式正则化效应：模型对某个样本不确定时，多个生成回答的奖励方差较大 → 自动降低梯度更新幅度 → 避免对旧知识的过度破坏。

这个机制像"自适应刹车"：不确定的地方慢更新，确定的地方快更新。

### 排除的假说

| 假说 | 实验结论 |
|------|---------|
| KL 散度惩罚项是恢复的主因 | 去掉 KL 惩罚，模型仍能恢复性能，但训练过程变不稳定 |
| CoT 推理机制是恢复的主因 | 去掉 CoT，模型照样保持泛化能力 |
| **真正的主因** | **奖励方差驱动的隐式正则化 + 权重方向重构** |

## 整体机制链

```
RL 训练初期
  → 参数空间被强行重塑
  → SFT 学到的特征对齐被打破
  → 测试集性能下降

RL 训练深入
  → 深层和浅层权重矩阵的奇异向量方向重新调整
  → 学习到更鲁棒的泛化模式
  → 新表示空间既保留高奖励能力，又恢复泛化能力
  → 性能回升，甚至超过 SFT 阶段水平
```

## 工程实践要点

### SFT 过度训练的代价

如果 SFT 训练时间过长、过度拟合训练数据，参数被拉得太远，RL 需要更多步数才能拉回来，有时甚至拉不回来。实践中很多团队在 SFT 中间 checkpoint 就开始 RL，不等完全收敛。

Llama-3.2-11B 的数据：SFT 训练 20% 时 OOD 性能就达到峰值，继续训练反而下降。

### RFT 天然抗遗忘

RFT（强化微调）相比 SFT，在持续学习场景下天然更不容易遗忘。SFT 会导致严重的灾难性遗忘（之前任务的性能大幅下降），但 RFT 即使不用数据回放策略，也能保持之前任务的性能，甚至接近多任务联合训练的效果。这说明 RL 训练范式本身对知识保留更友好。
