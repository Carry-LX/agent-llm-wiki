» 2.8 模式提炼
M Claude Code 的工具系统设计中，可以提炼出几个对 Al Agent 构建者普遍有价值的模式:
模式 1: 失败关闭的默认值。 buitLdTootL() 的默认值假设最危险的情况 〈不可并发、非只读) ，工具开发
者必须主动声明安全属性。这将安全从"选择加入"翻转为"选择退出"，大幅降低了遗漏导致的风险。
模式 2: 分层预算控制。 单工具结果有上限，单消息也有聚合上限。两层控制互相补充一一单工具上限防
止单点失控，消息上限防止并行调用的集体爆炸。
模式 3: 输入感知的属性。 isConcurrencySafe(input) 和 isReadOnly(input) 接收工具输入，而非全
局判断。同一个 BashTooL ，Ls 和 rm 有完全不同的安全属性。这种细粒度的输入感知是实现精确权限
控制的基础。详见第4章。
模式 4: 渐进泻染。 三阶段泻染 〈意图 一 进度 一 结果) 让用户在工具执行的每个阶段都有可见性。
Partial<Input> 的设计确保即使在参数流式传输期间，UI 也不会空白。这对用户信任至天重要一一用户
需要知道 Agent 正在做什么，而不是盯着一个旋转的加载图标。
模式 5: 编译期消除 vs 运行时过滤。 Feature Flag 通过 bun:bundle 的 feature() 在编译期消除未启用
的工具代码，而权限规则在运行时过滤工具列表。两种机制服务不同目的: 前者减小bundle 体积和攻击
面，后者支持用户级配置。
